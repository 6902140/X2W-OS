类型定义说明

◆ strap_handler_t

typedef int64_t(* strap_handler_t) (strapframe_t *stf_ptr)

strap_handler_t是异常/中断处理函数的函数类型, 函数原型为: int64_t funcname(struct strapframe_t*)

参数
stf_ptr指向被打断的程序的陷入帧, 程序的陷入帧将在strap_enter中构建在M模式的栈中
注解
如果被打断的程序想要传参给异常/中断处理函数, 那么必须要通过寄存器来实现, 而所有的通用寄存器在strap_enter函数中会被保存到陷入帧中. 所以异常/中断处理函数必然可以通过陷入栈来获得被打断的程序传入的参数
RISC-V通过a0~a7寄存器传参, 因此在strap_enter中要把strapframe的地址保存到a0寄存器中

◆ delegate_traps()

void delegate_traps ( void  )

delegate_traps用于将M模式下的一些中断和异常委托至S模式

注解
目前将以下中断/异常进行了委托:
  1. 软件中断, 时钟中断, 外部中断
  2. 指令未对齐异常, 读取指令页异常, 读取数据页异常, 存储数据页异常, 读取数据异常, 存储数据异常, 断点异常, U模式下的ecall异常
  1. 委托中断和异常本质上就是读写mideleg和medeleg寄存器. 这两个寄存器每一位对应一种中断/异常
  2. 设置mideleg和medeleg寄存器对应的位后, 对应的中断/异常发生后就会进入到ktrap_enter中
  3. 这两个寄存器每一位具体操作的是什么中断/异常, 参考RISC-V手册: 3.1.8 Machine Trap Delegation Registers (medeleg and mideleg)

◆ general_strap_handler()

NO_RETURN int64_t general_strap_handler ( strapframe_t *  stf_ptr)

general_strap_handler是所有中断/异常的缺省处理函数, 将会在屏幕上输出中断/异常信息, 打印陷入帧, 而后挂起内核

参数
stf_ptr保存被中断的程序上下文信息(所有通用寄存器和部分CSR寄存器)的陷入栈, 在strap_enter中构建
返回
NO_RETURN 该函数将挂起内核, 不会返回至内核继续运行

regitser_strap_handler是SBI异常/中断注册函数, 用于将编号为trap_code的异常/中断的处理函数strap_func注册到SBI的异常/中断处理函数表中

参数
trap_code注册的中断/异常的编号
interrupt若为True, 则将处理函数注册为中断处理函数, 否则注册为异常处理函数
msg中断的相关信息
strap_func中断/异常处理函数, 需要为类型strap_handler_t, 即函数原型需要为int64_t funcname(ireg_t mcause)
