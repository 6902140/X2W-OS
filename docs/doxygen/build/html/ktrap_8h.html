类型定义说明

◆ ktrap_handler_t

typedef int64_t(* ktrap_handler_t) (ktrapframe_t *ktf_ptr)

ktrap_handler_t是异常/中断处理函数的函数类型, 函数原型为: int64_t funcname(struct ktrapframe_t*)

参数
ktf_ptr指向被打断的程序的陷入帧, 程序的陷入帧将在ktrap_enter中构建在S模式的栈中
注解
如果被打断的程序想要传参给异常/中断处理函数, 那么必须要通过寄存器来实现, 而所有的通用寄存器在ktrap_enter函数中会被保存到陷入帧中. 所以异常/中断处理函数必然可以通过陷入栈来获得被打断的程序传入的参数
RISC-V通过a0~a7寄存器传参, 因此在ktrap_enter中要把ktrapframe的地址保存到a0寄存器中

◆ general_ktrap_handler()

NO_RETURN int64_t general_ktrap_handler ( ktrapframe_t *  ktf_ptr)

general_ktrap_handler是所有中断/异常的缺省处理函数, 将会在屏幕上输出中断/异常信息, 打印陷入帧, 而后挂起内核

参数
ktf_ptr保存被中断的程序上下文信息(所有通用寄存器和部分CSR寄存器)的陷入栈, 在ktrap_enter中构建
返回
NO_RETURN 该函数将挂起内核, 不会返回至内核继续运行

void regitser_ktrap_handler ( uint64_t  trap_code,
Bool  interrupt,
const char *  msg,
ktrap_handler_t  ktrap_func

regitser_ktrap_handler是SBI异常/中断注册函数, 用于将编号为trap_code的异常/中断的处理函数ktrap_func注册到SBI的异常/中断处理函数表中

参数
trap_code注册的中断/异常的编号
interrupt若为True, 则将处理函数注册为中断处理函数, 否则注册为异常处理函数
msg中断的相关信息
ktrap_func中断/异常处理函数, 需要为类型ktrap_handler_t, 即函数原型需要为int64_t funcname(ireg_t kcause)
